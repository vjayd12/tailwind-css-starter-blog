Are you excited to create your first `Maven-based AEM` project ? Here you go. We will create and deploy our AEM Maven-based project to an AEM instance. I hope you have your AEM instance up and running; if not, please check out the previous blog before we start with project creation.

Since we are close to project creation, I would like you to get some glimpse of Maven projects and Maven archetypes, which play a vital role for project creations.

![AEM Maven Project](https://miro.medium.com/v2/resize:fit:1400/format:webp/1*wmZVKl_cGHeLK7s_W0lpdQ.png)

## Steps for project creation and deployment :

**Step 1 :-** Open the terminal and check the maven version.

**Step 2 :-** Create the workspace for the project and execute the below command, which will create a Maven-based AEM project for you.

```javascript
mvn -B org.apache.maven.plugins:maven-archetype-plugin:3.2.1:generate \
  -DarchetypeGroupId=com.adobe.aem \
  -DarchetypeArtifactId=aem-project-archetype \
  -DarchetypeVersion=50 \
  -DappTitle="AEM Tutorial" \
  -DappId="aemtutorial" \
  -DgroupId="com.aemtutorial" \
  -DaemVersion=cloud \
  -DincludeDispatcherConfig=true \
  -DfrontendModule=general \
  -Dlanguage=en \
  -Dcountry=us \
  -DsingleCountry=false \
  -DincludeExamples=true \
  -DincludeErrorHandler=true
```

> [!Note]
> This is sample command for project you can create you own project as per the requirement. Check the properties from here and configure your project accordingly.

**Step 3 :-** After successfully creating the project, navigate to the project directory and enter the following command. This will build and deploy the whole project to an AEM instance.

```javascript
mvn clean install -PautoInstallSinglePackage
```

![code successfully deploy](https://miro.medium.com/v2/resize:fit:1400/format:webp/1*QPjQM6etp7Mr0oWjOhD67Q.png)

> [!Note]
> You can use `Eclipse / IntelliJ / Vscode` as IDE for code development.

We can run a few other commands too depending on what we want to deploy.

- If we are looking to deploy only apps, i.e., **frontend code**, then run the command

```javascript
mvn clean install -PautoInstallPackage
```

- If we want to deploy only **backend code** , i.e java code than run

```javascript
mvn clean install -PautoInstallBundle
```
